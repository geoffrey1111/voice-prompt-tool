# Voice Prompt Tool Desktop Package

这是本地语音转文本整理工具的 Windows 便携包。

## 启动

双击：

```text
启动 Voice Prompt Tool.exe
```

如果这个入口被杀毒软件拦截，也可以备用双击 `launch_desktop.bat`。

完整版自带便携 Python，不需要另外安装 Python。启动器会先检查包完整性、运行时和内存，并把启动过程写入 `logs/startup.log`。如果启动失败，命令行窗口会停住并显示原因。

启动后程序常驻后台，并立即开始加载本地 ASR 和 Qwen 模型。加载阶段会显示进度条；加载完成前不能录音，避免第一次使用时卡在识别或整理阶段。

- 按 `Ctrl+Space`：AI 整理模式。模型已就绪时，Windows 任务栏上方会显示一个小录音状态条；再按一次 `Ctrl+Space` 停止录音。停止后会先把原始识别文本插入当前输入框，并追加 `（处理前，整理中）`；本地 Qwen 整理完成后，会尽量只替换这次新增的那段文字。
- 按 `右 Alt`：快速听写模式。模型已就绪时，Windows 任务栏上方会显示一个小录音状态条；再按一次 `右 Alt` 停止录音。停止后只做语音识别和基础校对，不走 Qwen 整理，直接把结果插入当前输入框。

## 使用流程

1. 启动后等待模型加载完成。加载窗口会显示当前阶段和进度条。
2. 按 `Ctrl+Space` 开始 AI 整理录音，或按 `右 Alt` 开始快速听写。
3. 录音时只显示一个贴近任务栏上方的小状态条；再次按同一个快捷键后停止录音。处理阶段状态条会显示 `识别中` 或 `整理中`，不再继续显示录音计时。
4. 停止后，本地 SenseVoiceSmall 会识别语音。
5. AI 整理模式会继续调用本地 Qwen3 4B Instruct 整理口述文本，并替换刚刚插入的 `处理前` 文本。
6. 如果当前输入框无法安全插入或替换，程序会展开结果窗口；上方两个文本框都可以手动编辑，点击复制按钮会复制当前文本框内容并关闭窗口。

## 托盘菜单和设置

程序运行时会留在 Windows 任务栏托盘区。右键托盘图标可以：

- 开始录音并整理
- 快速听写
- 打开设置
- 预热模型
- 释放模型
- 打开日志目录
- 退出

设置里可以开启开机自启动、设置空闲多久释放模型，以及选择整理强度：忠实整理、简洁整理、强理解转述。当前版本固定在启动后加载模型，加载完成后才允许录音。默认使用强理解转述，让整理后的文本更像重新理解后写出的完整表达。

如果设置了空闲自动释放模型，释放时会弹出托盘提示，窗口状态也会显示模型已释放。释放后再次按快捷键不会直接录音，而是先进入模型加载状态；加载完成后再使用快捷键录音。

## 本地与隐私

- 默认不调用云服务。
- 语音识别使用包内 SenseVoiceSmall。
- 文本整理使用包内 Ollama + `qwen3:4b-instruct`。
- 录音开始时会在后台预热 SenseVoice 和 Ollama/Qwen，减少停止录音后的首次等待。
- 桌面程序退出或点击托盘里的 `释放模型` 时，会关闭本工具包内的 Ollama/llama-server 进程。

## 文件位置

- 录音文件保存在包内 `recordings` 目录。
- 日志文件保存在包内 `logs` 目录。
- 模型文件已随包放在包内，不需要每次下载。

## 注意

如果 `Ctrl+Space` 被系统输入法或其他软件占用，桌面程序会提示快捷键注册失败。关闭占用该快捷键的软件后重新启动本工具。

如果双击后没有看到窗口，或者怀疑旧进程没有关干净，先双击：

```text
stop_desktop.bat
```

然后再双击 `启动 Voice Prompt Tool.exe`。如果 exe 被拦截，再尝试 `launch_desktop.bat`。

如果仍然打不开，把下面这个文件发给开发者排查：

```text
logs\startup.log
```

常见原因包括：压缩包没有完整解压、杀毒软件拦截、内存不足、第一次下载模型失败，或者快捷键被其他软件占用。
